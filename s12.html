Semana 12

1. Arquitectura de Aplicaciones Web

1. Concepto: En el desarrollo backend, la arquitectura de aplicaciones web sigue patrones como el Modelo-Vista-Controlador (MVC) o el Modelo-Vista-Servicio (MVS). En el contexto de Jakarta EE, el diseño modular y escalable es clave. La lógica de negocios reside en los EJBs (Enterprise JavaBeans), y los Servlets y JSPs (JavaServer Pages) manejan la interacción con el cliente.

2. Servidores Web y Aplicaciones

1. Concepto: Los servidores web gestionan solicitudes HTTP y sirven contenido estático, mientras que los servidores de aplicaciones, como Apache Tomcat o WildFly, son responsables de ejecutar la lógica del servidor. Jakarta EE es un estándar para la construcción de aplicaciones empresariales en Java y se implementa en servidores de aplicaciones compatibles.

3. Funcionamiento del Server-Side

1. Concepto: En el server-side, las solicitudes del cliente se manejan en el servidor, ejecutando la lógica de negocio y generando respuestas dinámicas. Jakarta EE proporciona un conjunto robusto de API y componentes para simplificar el desarrollo de aplicaciones empresariales, gestionando aspectos como la transaccionalidad, seguridad, y la conexión a bases de datos.

4. Lenguaje y Framework Backend

1. Concepto: Java es un lenguaje backend muy popular debido a su portabilidad, orientación a objetos, y robustez. Jakarta EE (anteriormente Java EE) es un conjunto de especificaciones y APIs para el desarrollo de aplicaciones empresariales en Java. Proporciona un conjunto de servicios comunes y está diseñado para ser escalable y modular.

5. Entorno de Desarrollo - IntelliJ IDEA

1. Concepto: IntelliJ IDEA es un entorno de desarrollo integrado (IDE) altamente utilizado para Java y Jakarta EE. Ofrece características avanzadas como la finalización de código, depuración, refactorización, y soporte para frameworks populares. Su interfaz intuitiva y funciones avanzadas facilitan el desarrollo de aplicaciones backend de manera eficiente.

6. Servidor de Aplicaciones - Apache Tomcat

1. Concepto: Apache Tomcat es un servidor de aplicaciones Java que implementa las tecnologías Java Servlet, JavaServer Pages y Java Expression Language. Es liviano y fácil de configurar, siendo una elección común para implementar aplicaciones Jakarta EE.

7. Terminología y Características de Spring

1. Concepto: Spring es un marco de desarrollo de aplicaciones para Java que proporciona una infraestructura integral para el desarrollo de aplicaciones empresariales. Algunos términos y características clave incluyen:

* Inyección de Dependencias (DI): Facilita la gestión de dependencias y promueve la inversión de control.

* Aspect-Oriented Programming (AOP): Permite modularizar aspectos transversales como la seguridad y el registro.

* Contenedor de Spring: Gestiona y controla los objetos de la aplicación, proporcionando funcionalidades de configuración y servicios.

8. Módulos de Spring

1. Concepto: Spring se organiza en módulos, cada uno centrado en aspectos específicos del desarrollo. Algunos módulos notables son:

* Spring Core Container: Proporciona el contenedor IoC y DI.

* Spring AOP: Implementa AOP para modularizar aspectos.

* Spring JDBC y ORM: Facilita la integración con bases de datos.

* Spring MVC: Ofrece un marco para construir aplicaciones web basadas en el patrón MVC.

8. Servlets y JSP

1. Servlets: Son clases Java que extienden la funcionalidad de un servidor web para manejar solicitudes y respuestas HTTP. Spring MVC utiliza Servlets para gestionar las peticiones web.

2. JSP(JavaServer Pages): Son páginas web Java que contienen código Java embebido en HTML. Spring MVC utiliza JSP para las vistas web dinámicas.

9. Spring Boot

1. Concepto: Spring Boot simplifica drásticamente el desarrollo de aplicaciones Spring al proporcionar configuraciones predeterminadas y convenciones sensatas. Algunas características destacadas incluyen:

* Embalaje Independiente: Las aplicaciones Spring Boot pueden empaquetarse como archivos JAR autocontenidos.

* Autoconfiguración: Spring Boot intenta configurarse automáticamente según las dependencias en tu proyecto.

* Spring Boot Starters: Conjuntos de dependencias predeterminadas para simplificar la configuración.
