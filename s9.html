Semana 9

Next.js es un marco de desarrollo web de React que facilita la creación de aplicaciones web modernas y eficientes. Ofrece varias características poderosas, incluyendo la capacidad de elegir entre la renderización del lado del cliente (CSR) y del lado del servidor (SSR) para optimizar el rendimiento y la experiencia del usuario

1. Renderización del lado del Cliente (CSR)

1. Concepto: En este enfoque, la carga inicial de la página ocurre en el navegador del usuario. La aplicación React se ejecuta en el navegador, y la interfaz de usuario se construye dinámicamente en el cliente.

2. Ventajas:

Rápida navegación inicial: Las páginas se cargan rápidamente, ya que solo se envía el código JavaScript necesario y los datos se solicitan después.

Interactividad del cliente: Ideal para aplicaciones que requieren mucha interactividad del usuario.

3. Implementación en Next.js: Para realizar CSR en Next.js, simplemente construyes tus páginas utilizando componentes React y las exportas como se hace comúnmente.

2. Renderización del lado del Servidor (SSR)

1. Concepto: En este enfoque, la página se genera en el servidor y se envía al navegador como HTML estático. Esto significa que el usuario recibe una página ya renderizada.

2. Ventajas:

Mejor SEO: Los motores de búsqueda pueden indexar el contenido directamente, ya que reciben una página completamente renderizada.

Mejor redimiento para la primera carga: Usuarios ven el contenido más rápido ya que reciben una página completa.

3. Implementación en Next.js: Para realizar SSR en Next.js, exportas una función getServerSideProps desde tu componente.

3. Elección entre CSR y SSR en Next.js

1. CSR: Ideal para aplicaciones donde la interactividad del cliente es clave y la velocidad de carga inicial es prioritaria.

2. SSR: Adecuado para aplicaciones que requieren un mejor rendimiento de SEO y una carga inicial más rápida.
