Semana 13

1. Introducción a API REST

1. Concepto: Una API REST (Representational State Transfer) es una interfaz de programación de aplicaciones que sigue los principios y restricciones de la arquitectura REST. Se basa en operaciones HTTP como GET, POST, PUT y DELETE para realizar operaciones CRUD (Create, Read, Update, Delete) en recursos.

2. Protocolo HTTP

1. Verbos HTTP:

* GET: Recupera información.

* POST: Crea un nuevo recurso.

* PUT: Actualiza un recurso existente.

* DELETE: Elimina un recurso.

2. Códigos de Estado HTTP:

* 200 OK: Éxito en la solicitud.

* 201 Created: Recurso creado con éxito.

* 204 No Content: Éxito sin contenido.

* 400 Bad Request: Error en la solicitud del cliente.

* 401 Unauthorized: Falta de autentitación.

* 404 Not Found: Recurso no encontrado.

* 500 Internal Server Error: Error interno del servidor.

3. MVC y Arquitectura de API REST

1. MVC (Modelo-Vista-Controlador)

* Modelo: Representa los datos y la lógica del negocio.

* Vista: Presenta la información al usuario.

* Controlador: P Maneja las solicitudes y gestiona la lógica de negocio.

2. Arquitectura de API REST con Spring

* @RestController: Anotación de Spring para marcar clases como controladores REST.

* @RequestMapping: Define la ruta base para las operaciones del controlador.

* DTO (Data Transfer Object): Modela la estructura de datos transferida entre el cliente y el servidor.

4. Desarrollo de API REST con Spring Boot

1. MVC (Modelo-Vista-Controlador): Selecciona Spring Web al configurar tu proyecto.

5. Pruebas con Postman

1. Postman: Herramienta para probar API REST. Crea solicitudes HTTP y verifica las respuestas del servidor.

6. Seguridad en API REST

1. Spring Security: Proporciona características de seguridad para proteger las API REST.

2. Token JWT (JSON Web Token): Utilizado para autenticación y autorización
