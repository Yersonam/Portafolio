Semana 2

Para desarrollar una página web, se necesita de 2 partes, la primera que es la parte estructural que para esto se usa HTML y la segunda que viene a ser el diseño que se puede realizar con CSS. HTML como vimos en la semana pasada es la base de la estructura de una página, en esta tecnología se utilizan etiquetas para organizar el contenido de manera semántica.
Por último, CSS se utiliza para dar estilo y diseño a la página, pues con esta tecnología se define el color, tamaño, tipografía, disposición y otros aspectos visuales. Con CSS se puede aplicar tanto a elementos individuales como a clases y etiquetas específicas, utilizado reglas de estilo.
Por otro lado, es importante mencionar que a estas 2 de tantas tecnologías lo engloba el Estándar Open Web Plataform.

1. Estándar Open Web Plataform:

Etiquetas HTML: En HTML (HyperText Markup Language), las etiquetas son elementos fundamentales que se utilizan para estructurar y dar formato al contenido dentro de un documento web. Las etiquetas HTML son piezas de código encerradas entre corchetes angulares < > que indican al navegador cómo mostrar o interpretar el contenido que está entre ellas. Cada etiqueta tiene un propósito específico y define diferentes partes de un documento web.

Ejemplos de etiquetas comunes de HTML:

< p >: Define un párrafo.
< h1 > ... < h6 >: Definen encabezados de diferentes niveles.
< a >: Crea un enlace a otra página web o recurso.
< img >: Inserta una imagen.
< ul >, < ol >, < li >: Define listas desordenadas (viñetas), listas ordenadas (números) y elementos de lista, respectivamente.

Estructura HTML: La estructura básica de un documento HTML (HyperText Markup Language) sigue un esquema estándar que incluye elementos fundamentales para crear una página web.

Esta estructura básica consta de varias secciones:

< !DOCTYPE html >: Declaración que especifica la versión de HTML que se está utilizando (en este caso, HTML5).
< html lang="es" >: Elemento raíz que envuelve todo el contenido HTML, con el atributo lang para indicar el idioma principal de la página (en este caso, español).
< head >: Sección que contiene metadatos, enlaces a estilos CSS, scripts, y la configuración general de la página. También incluye el título que se muestra en la pestaña del navegador.
< body >: Contiene todo el contenido visible de la página web, como encabezados, párrafos, imágenes, listas, etc.
< header >, < main >, < footer >: Secciones semánticas que ayudan a estructurar y organizar el contenido de la página. < header > representa el encabezado (logo, menú, etc.), < main > el contenido principal y < footer > el pie de página.

2. Árbol DOM

El Árbol de Documento del Modelo de Objetos del Documento (DOM por sus siglas en inglés, Document Object Model) es una representación jerárquica de un documento HTML o XML. Este modelo es la estructura de objetos generada por el navegador cuando interpreta un documento y representa los elementos, atributos y relaciones entre ellos como un árbol de nodos. El DOM comienza con un nodo raíz que representa todo el documento y se ramifica en diferentes nodos que representan elementos, atributos, texto y otros contenidos del documento. Cada elemento HTML se convierte en un nodo en este árbol, lo que permite a los desarrolladores acceder, modificar y manipular dinámicamente el contenido, la estructura y los estilos de la página web mediante scripting (como JavaScript).

Árbol de Nodos: El árbol de nodos se refiere a la estructura jeráquica de nodos que constituyen el DOM en HTML. Cada elemento, atributo, comentario o fragmento de texto en un documento HTML es representado por un nodo en este árbol.

Los nodos en el DOM tienen diferentes tipos y se organizan en una estructura de árbol. Algunos de los tipos de nodos más comunes son:

Element Node (Nodo de Elemento): Representa una etiqueta HTML en el árbol DOM. Por ejemplo, un nodo < div >, < p >, < a >, etc.
Text Node (Nodo de Texto): Contiene el texto dentro de un elemento HTML. Por ejemplo, el contenido textual entre etiquetas < p >Texto aquí< /p > se representa como un nodo de texto.
Comment Node (Nodo de Comentario): Representa un comentario HTML < !-- Comentario -- > en el documento.
Document Node (Nodo de Documento): Es el nodo raíz del árbol DOM, que representa todo el documento HTML.
Attribute Node (Nodo de Atributo): Representa los atributos de un elemento HTML.

Cada nodo en este árbol representa un componente específico del documento HTML y sus relaciones jerárquicas. El conocimiento y manipulación de estos nodos son fundamentales para interactuar y modificar dinámicamente el contenido de una página web utilizando JavaScript u otras tecnologías del lado del cliente.

Renderizado de Árbol: El proceso de renderizado de un árbol en el contexto de desarrollo web se refiere a la representación visual de la estructura jerárquica de nodos en una página web. En términos simples, este proceso involucra la interpretación de la estructura del árbol de nodos para mostrar el contenido en un navegador o en un entorno donde se presente la página web.

El proceso de renderizado implica varios pasos:

Construcción del Árbol DOM: El navegador interpreta el código HTML y crea el Árbol de Nodos (DOM) correspondiente, representando la estructura jerárquica de todos los elementos, atributos y texto del documento.
Estilo y Diseño (Cálculo de Estilos): Una vez que se ha construido el DOM, el navegador calcula los estilos asociados a cada nodo, teniendo en cuenta las hojas de estilos CSS y otras reglas aplicables. Esto determina cómo se verán los elementos en la página, incluyendo cosas como el tamaño, color, posición, entre otros.
Disposición (Layout): Después de calcular los estilos, el navegador realiza el cálculo del diseño de la página. Esto implica determinar la posición y tamaño de cada elemento en relación con otros elementos en la página. Se decide dónde se colocarán los elementos y cómo se distribuirá el espacio disponible.
Renderizado (Pintado en la Pantalla): En esta etapa, el navegador dibuja los elementos en la pantalla basándose en la información calculada anteriormente. Se pinta cada píxel de la ventana del navegador de acuerdo con las especificaciones proporcionadas por el árbol de nodos, los estilos y el diseño calculado.
Attribute Node (Nodo de Atributo): Representa los atributos de un elemento HTML.

Este proceso de renderizado ocurre cada vez que se carga una página web o cuando se modifican elementos a través de JavaScript u otros medios que puedan afectar la presentación del contenido.

Ciclo de Vida de una Página Web: El ciclo de vida de una página web describe las diferentes etapas que experimenta una página desde su creación hasta su visualización completa en el navegador, así como los eventos y procesos que ocurren en cada una de estas etapas.

A continuación se detallan las principales etapas del ciclo de vida de una página web:

Solicitud y carga: Todo comienza cuando un usuario ingresa la URL en el navegador o hace clic en un enlace. El navegador envía una solicitud al servidor web para obtener el recurso (archivo HTML, CSS, imágenes, scripts, etc.) asociado con esa URL.
Procesamiento en el servidor: El servidor recibe la solicitud, procesa la petición y envía de vuelta al navegador el archivo HTML correspondiente, junto con otros recursos necesarios (como archivos CSS, JavaScript, imágenes, etc.).
Construcción del DOM: Una vez que el navegador recibe el HTML, comienza a interpretarlo y construye el Modelo de Objetos del Documento (DOM), es decir, la representación en forma de árbol de la estructura del documento.
Carga de recursos adicionales:El navegador continúa cargando los recursos adicionales que se encuentran en el HTML, como hojas de estilo (CSS), scripts (JavaScript), imágenes y otros archivos multimedia. Cada recurso solicitado puede desencadenar una nueva petición al servidor.
Procesamiento CSS y JavaScript:Después de cargar los recursos adicionales, el navegador analiza y ejecuta los archivos CSS y JavaScript. Esto incluye el cálculo de estilos, la ejecución de scripts y la manipulación del DOM.
Renderizado:Una vez que se ha procesado todo el contenido y se han aplicado los estilos, el navegador procede al renderizado, donde pinta los elementos en la pantalla del usuario siguiendo el diseño y la disposición calculados.
Interacción del usuario: Una vez que la página se ha cargado por completo, el usuario puede interactuar con ella. Esto puede implicar clics en enlaces, botones, formularios, desplazamientos y cualquier otra acción que genere eventos en la página.
Actualizaciones y cambios dinámicos:Durante la interacción del usuario, pueden ocurrir cambios en la página, como la modificación del contenido a través de JavaScript, la presentación de animaciones, la carga de contenido adicional mediante solicitudes AJAX, entre otros.
Descarga y finalización:Cuando el usuario sale de la página o navega a otra, se liberan los recursos asociados con esa página. Esto puede incluir la eliminación de los objetos del DOM, la descarga de recursos en caché y otras tareas de limpieza.

Optimización en motores de búsqueda SEO: La optimización para motores de búsqueda (SEO, por sus siglas en inglés) es un conjunto de prácticas destinadas a mejorar la visibilidad y el posicionamiento de un sitio web en los resultados de búsqueda orgánica de motores como Google, Bing o Yahoo.

3. Diseño

Estándar CSS3: Es la última versión del estándar CSS. CSS3 ha introducido muchas características nuevas y mejoras con respecto a las versiones anteriores de CSS, permitiendo a los desarrolladores web estilizar y diseñar páginas web de manera más sofisticada y efectiva.

Algunas de las características destacadas de CSS3 incluyen:

Selectores avanzados: CSS3 ha ampliado la capacidad de seleccionar elementos de manera más específica y compleja utilizando selectores avanzados como nth-child, nth-of-type, adjacent sibling selectors y más.
Propiedades de bordes y esquinas: Introdujo propiedades como border-radius para crear esquinas redondeadas en elementos, box-shadow para sombras y border-image para utilizar imágenes en lugar de bordes simples.
Gradients: Permite la creación de fondos con degradados mediante la propiedad linear-gradient o radial-gradient, permitiendo transiciones suaves de color.
Transiciones y animaciones: CSS3 facilita la creación de transiciones suaves entre los estados de los elementos (hover, active, etc.) utilizando transition, así como la animación de elementos mediante la propiedad @keyframes.
Fuentes personalizadas: La capacidad de usar fuentes personalizadas se ha mejorado con @font-face, lo que permite a los diseñadores utilizar fuentes no estándar en sus diseños web.
Media Queries: CSS3 introduce Media Queries, lo que permite a los desarrolladores adaptar el diseño y estilo de una página web en función del dispositivo en el que se visualiza, como en pantallas móviles, tablets o desktops.
Flexbox y Grid: Introdujo módulos como Flexbox y CSS Grid Layout, que permiten un diseño más flexible y avanzado en la disposición de elementos en una página web, facilitando la creación de diseños responsivos y complejos.
