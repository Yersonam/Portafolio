Semana 3

Bootstrap es un Framework de HTML, CSS y JavaScript utilizado para el desarrollo de interfaces de usuario web responsivas y móviles. Fue creado por Twitter y ha sido ampliamente adoptado por desarrolladores web debido a su facilidad de uso, flexibilidad y amplia gama de componentes preestilizados. En esta semana comentaremos acerca de 2 frameworks: Bootstrap y Tailwind.

1. Características clave de Bootstrap:

Grid system: Ofrece un sistema de rejilla (grid system) que permite crear diseños flexibles y adaptables a diferentes tamaños de pantalla, facilitando el diseño responsive.

Componentes UI preestilizados: Proporciona una amplia variedad de componentes preestilizados, como botones, formularios, carruseles, menús desplegables, navegación, tarjetas, entre otros, que pueden ser utilizados fácilmente y personalizados según las necesidades del proyecto.

Diseño responsivo: Bootstrap está diseñado para crear sitios web responsivos, lo que significa que los sitios creados con Bootstrap se ajustan y se ven bien en dispositivos de diferentes tamaños, desde teléfonos móviles hasta computadoras de escritorio.

Documentación extensa: Ofrece una documentación detallada, ejemplos de código y directrices para ayudar a los desarrolladores a implementar rápidamente componentes y estilos proporcionados por Bootstrap.

Compatibilidad con navegadores: Bootstrap se esfuerza por ser compatible con los navegadores más comunes, asegurando que los sitios web creados con esta librería funcionen de manera consistente en diferentes entornos de navegación.

2. Layout

Se refiere a la estructura de los formularios.

Forms

Cada grupo de campos de formulario debe residir en un elemento < form >. Bootstrap no proporciona ningún estilo por defecto para el elemento < form >, pero hay algunas potentes características del navegador que se proporcionan por defecto.

* Los < button > dentro de un < form > tienen por defecto 'type'="submit", así que es necesario ser específico e incluir siempre un tipo.
* Contiene el texto dentro de un elemento HTML. Por ejemplo, el contenido textual entre etiquetas < p >Texto aquí< /p > se representa como un nodo de texto.
* Se puede desactivar cada elemento de formulario dentro de un formulario con el atributo disabled en el < form >.
* Bootstrap aplica display: block y width: 100% a casi todos los controles de formulario, por esta razón los formularios se apilarán por defecto verticalmente, sin embargo, se pueden utilizar clases adicionales para variar este diseño en cada formulario.

Utilities

Son la forma más sencilla de añadir estructura a los formularios. Proporcionan una agrupación básica de etiquetas, controles, texto opcional del formulario y mensajes de validación del formulario. Recomendamos ceñirse a las utilidades de margen-inferior y utilizar una única dirección en todo el formulario para mantener la coherencia.

Form grid

Se pueden crear formularios más complejos utilizando las clases de rejilla. Se recomienda utilizarlas para diseños de formularios que requieran múltiples columnas, anchos variados y opciones de alineación adicionales. Asimismo, se requiere que la variable $enable-grid-classes de Sass esté activada (activada por defecto).

Grutters

Column sizing

El sistema de rejilla permite colocar cualquier número de .cols dentro de una .row. Asimismo, estas clases dividirán el ancho disponible a partes iguales entre ellas. También se puede elegir un subconjunto de columnas para que ocupen más o menos espacio, mientras que las .cols restantes se reparten el resto a partes iguales, con clases de columna específicas como .col-sm-7.

3. Components

Bootstrap, como framework de desarrollo web, ofrece una amplia gama de componentes preestilizados y listos para usar que facilitan la creación de interfaces de usuario modernas y receptivas.

Algunos de los componentes más comunes incluyen:

Botones (Buttons): Ofrece estilos predefinidos para botones, como botones de acción, botones de enlace, botones de diferentes tamaños y estilos, así como botones de grupos y barras de herramientas.
Formularios (Forms): Proporciona estilos para diferentes tipos de formularios, incluyendo campos de texto, áreas de texto, casillas de verificación, botones de radio, menús desplegables y elementos de formulario personalizados.
Navs y Navbar (Navs y Navbar): Ofrece estilos para crear barras de navegación responsivas, menús de pestañas (tabs), menús de navegación (navbars) con funcionalidad colapsable para dispositivos móviles, entre otros.
Alertas (Alerts): Componentes para mostrar mensajes de alerta, como mensajes de éxito, advertencia, información o errores.
Tarjetas (Cards): Componentes versátiles para mostrar contenido en tarjetas, que pueden contener imágenes, texto, encabezados, pies de página y más, facilitando la presentación de información de manera organizada y atractiva.
Carrousel (Carousel): Un componente interactivo que permite mostrar una serie de elementos (como imágenes o contenido) en una secuencia deslizable.
Modal (Modal): Ofrece la funcionalidad para crear ventanas modales emergentes que pueden mostrar contenido adicional, solicitar información al usuario o mostrar mensajes importantes.
Dropdowns (Dropdowns): Permite crear menús desplegables y elementos con múltiples opciones que se muestran al hacer clic o pasar el cursor sobre un elemento específico.
Progress Bars (Progress): Componentes para mostrar barras de progreso y barras de carga que indican el progreso de una tarea o proceso.
Tabs (Tabs): Estilos predefinidos para crear pestañas de navegación que permiten mostrar múltiples conjuntos de contenido en el mismo espacio.

4. Framework Tailwind

Tailwind CSS es un framework de desarrollo web que se centra en brindar utilidades y clases predefinidas que permiten crear interfaces de usuario altamente personalizadas y responsivas. A diferencia de otros frameworks como Bootstrap, que ofrecen componentes preestilizados, Tailwind proporciona un conjunto extenso de clases CSS predefinidas que se pueden utilizar directamente en el marcado HTML para estilizar y diseñar los elementos de la interfaz.

Algunas características y principios clave de Tailwind CSS incluyen:

Clases utilitarias: En lugar de ofrecer componentes prefabricados, Tailwind se basa en clases utilitarias como bg-gray-200 (para el color de fondo gris), text-blue-500 (para el color del texto azul), p-4 (para relleno) y muchas otras que se aplican directamente en las etiquetas HTML para estilizar los elementos.
Diseño responsivo: Proporciona clases responsivas que permiten definir estilos específicos para diferentes tamaños de pantalla, como sm (pequeño), md (mediano), lg (grande) y xl (extra grande).
Personalización y extensibilidad: Tailwind es altamente personalizable y permite a los desarrolladores crear sus propias utilidades y clases personalizadas, lo que facilita la creación de estilos consistentes y específicos para un proyecto en particular.
Configurabilidad: Se puede configurar mediante un archivo de configuración que permite ajustar colores, tipografía, márgenes, rellenos y otras configuraciones para adaptarse a las necesidades del proyecto.
Bajo nivel de abstracción: A diferencia de otros frameworks que ocultan el CSS detrás de componentes preestilizados, Tailwind expone explícitamente el CSS, lo que brinda un mayor control y visibilidad sobre el código generado.
Legibilidad y mantenibilidad: Utiliza nombres de clases descriptivos y coherentes, lo que facilita la legibilidad del código y la comprensión de los estilos aplicados a los elementos.
