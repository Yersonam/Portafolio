Semana 5

El Client-Side Rendering (CSR) o Renderizado en el Lado del Cliente es un enfoque utilizado en el desarrollo web donde la mayoría o la totalidad del procesamiento necesario para renderizar la interfaz de usuario se realiza en el navegador del usuario, utilizando JavaScript.

1. Funcionamiento

Descarga inicial: Cuando un ususario accede a una aplicación web que utiliza CSR, el servidor envía un HTML básico junto con los archivos de JavaScript y CSS necesarios.

Carga y ejecución del código: El navegador del usuario descarga estos archivos JavaScript que contienen la lógica de la aplicación. Una vez descargados, se ejecutan en el navegador del cliente.

Generación del DOM: El código JavaScript se encarga de generar la estructura del DOM (Document Object Model) basada en los datos y la lógica de la aplicación.

Interactividad dinámica: Las interacciones del usuario, como clics en botones o cambios en los datos, desencadenan actualizaciones en el DOM. Esto puede ocurrir de forma dinámica sin necesidad de recargar toda la página.

2. Ventajas

Experiencia de usuario interactiva: Permite interfaces de usuario más interactivas y dinámicas, ya que las actualizaciones se realizan rápidamente sin necesidad de recargar la página completa.

Separación de preocupaciones: Facilita la separación entre el frontend y el backend, permitiendo que los desarrolladores trabajen de forma más independiente en cada capa.

Mejora en el rendimiento percibido: A pesar de la descarga inicial de recursos, una vez cargados, las actualizaciones de datos se realizan de manera rápida y suave.

3. Desafíos

SEO (Optimización para Motores de Búsqueda): Los motores de búsqueda pueden tener dificultades para indexar contenido generado dinámicamente, lo que puede afectar el SEO. Sin embargo, las técnicas modernas como el prerrenderizado o el uso de técnicas de server-side rendering pueden abordar este problema.

Tiempo de carga inicial: La descarga inicial de JavaScript y recursos puede ralentizar el tiempo de carga inicial, especialmente en dispositivos con conexiones lentas.

Manipulación del DOM: Modificar la estructura y el contenido de la página web en tiempo real.

Problemas de accesibilidad: Si no se maneja adecuadamente, puede haber problemas de accesibilidad para usuarios con necesidades especiales o dispositivos con recursos limitados.

4. Desarrollo basado en componentes

El desarrollo basado en componentes, junto con conceptos como JSX, Props y Children, es una parte fundamental en el desarrollo de aplicaciones modernas utilizando frameworks como React. Estos conceptos permiten la creación de interfaces de usuario modulares y reutilizables.

Aquí explico brevemente cada uno de ellos:

JSX (JavaScript XML): JSX es una extensión de JavaScript que permite escribir código similar a HTML dentro de JavaScript. Facilita la creación de interfaces de usuario dentro de componentes de React.
Props: Las Props son la forma de pasar datos de un componente a otro en React. Son un mecanismo de solo lectura, lo que significa que el componente padre puede pasar datos al componente hijo, y el hijo no puede modificar esas props.
Children: La propiedad children es una prop especial en React que permite pasar contenido anidado a un componente. Esta prop contiene todo lo que está dentro de las etiquetas del componente al que se le pasa como hijo.
