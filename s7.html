Semana 7

Los hooks permite a los desarrolladores utilizar el estado y otras características en componentes funcionales, sin necesidad de convertirlos en componentes de clase. Los hooks son funciones especiales proporcionadas que permiten "engancharse" a características internas de como el estado, el ciclo de vida, el contexto, entre otros.

1. Usos comunes

useState(): El hook useState se utiliza para agregar estado a los componentes funcionales. Permite al componente mantener su propio estado interno. La función useState retorna un par de valores: el estado actual y una función para actualizar ese estado.

useEffect: El hook useEffect permite realizar efectos secundarios en componentes funcionales. Se ejecuta después de que el componente se monta o actualiza. Se usa para manipular el DOM, suscripciones a eventos, peticiones a APIs, entre otros.

useContext: El hook useContext permite acceder al contexto en componentes funcionales. Proporciona una forma de acceder a datos globales o compartir datos entre componentes sin necesidad de pasar props manualmente a través de varios niveles de componentes.

useReducer: El hook useReducer es una alternativa a useState para manejar estados complejos. Se utiliza para manejar estados que implican lógica más compleja y cuando el estado tiene múltiples subvalores o una lógica de actualización más sofisticada.

useRef: El hook useRef crea un objeto mutable que persiste a lo largo de los ciclos de renderizado del componente. Se utiliza principalmente para acceder al DOM o para mantener referencias a valores que persisten entre renderizaciones.

useCallback: El hook useCallback devuelve una versión memorizada de una función, lo que significa que la función no se vuelve a crear en cada renderizado, a menos que las dependencias cambien. Se usa para evitar la creación innecesaria de funciones en componentes que pueden causar renders adicionales.

useMemo: El hook useMemo memoriza el resultado de una función y solo se recalcula cuando una de las dependencias cambia. Se utiliza para evitar cálculos costosos durante las renderizaciones si las entradas no han cambiado.
