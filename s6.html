Semana 6

Los eventos en el contexto del desarrollo web son acciones o sucesos que ocurren en el navegador del usuario y que pueden ser detectados y manejados mediante JavaScript u otros lenguajes de programación en el frontend. Estos eventos pueden ser desencadenados por diversas acciones, como clics del mouse, presionar teclas, cargar una página, cambios en el tamaño de la ventana, entre otros.

1. Características

Interacción del usuario: La mayoría de los eventos están relacionados con la interacción del usuario con la interfaz de la aplicación web.

Desencadenados por acciones específicas: Cada tipo de evento se activa en respuesta a una acción particular, como hacer clic en un botón, mover el mouse sobre un elemento, escribir en un campo de texto, etc.

Captura y respuesta: Los eventos se pueden capturar mediante scripts y se pueden manejar para ejecutar ciertas acciones o desencadenar funciones específicas en respuesta a la acción del usuario.

2. Ejemplos comunes

Eventos del mouse: click, mouseover, mouseout, mousemove, etc.

Eventos de teclado: keydown, keyup, keypress, etc.

Eventos de formulario: submit, change, input, etc.

Eventos de carga: load, DOMContentLoaded, beforeunload, etc.

Eventos de ventana: resize, scroll, focus, blur, etc.

2. Renderizado Condicional

El renderizado condicional es una técnica utilizada en el desarrollo de aplicaciones web para mostrar u ocultar elementos en función de ciertas condiciones lógicas. Esto es especialmente útil para crear interfaces dinámicas que se adapten a diferentes estados o datos.

3. Renderizado Iterativo

El renderizado iterativo, también conocido como renderizado de listas, es una técnica común utilizada en el desarrollo de aplicaciones web para representar elementos de manera dinámica a partir de datos de lista, como matrices o conjuntos. Esto es especialmente útil cuando se desea renderizar múltiples elementos basados en datos.

3. Formularios

Los formularios son una parte fundamental de las aplicaciones web y permiten a los usuarios enviar datos al servidor o interactuar con la interfaz de la aplicación. En HTML, los formularios se crean utilizando la etiqueta < form > y pueden contener varios elementos de entrada, como campos de texto, selectores, botones y áreas de texto.

4. Routing

El enrutamiento es un concepto clave en el desarrollo web que permite navegar entre diferentes vistas o páginas dentro de una aplicación web sin necesidad de recargar la página completa. Los sistemas de enrutamiento son fundamentales para crear aplicaciones de una sola página (SPA) y permiten una experiencia de usuario más fluida al cambiar dinámicamente el contenido mostrado en función de la URL.

5. Consumo de APIs

Son conjuntos de reglas, protocolos y herramientas que permiten que diferentes aplicaciones se comuniquen entre sí. Estas interfaces permiten que distintos sistemas informáticos se conecten y compartan datos, funcionalidades o servicios de manera estandarizada y controlada.

Características

Interoperabilidad: Las APIs permiten que diferentes sistemas y aplicaciones, independientemente de su lenguaje de programación o arquitectura, se comuniquen y trabajen juntos.
Abstracción: Las APIs ocultan la complejidad interna de un sistema, ofreciendo una interfaz simplificada y consistente para interactuar con él.
Reutilización: Las APIs permiten el acceso a servicios o datos específicos sin necesidad de recrearlos, fomentando la reutilización y la eficiencia en el desarrollo de software.

Tipos

APIs Web: Son APIs expuestas por aplicaciones web para permitir la interacción con sus servicios o datos. Pueden ser RESTful, SOAP, GraphQL, entre otros.
APIs de Sistema Operativo: Permiten a las aplicaciones acceder a funcionalidades del sistema operativo, como las APIs de Windows, Android, iOS, etc.
APIs de Bibliotecas y Frameworks: Proporcionan funcionalidades específicas dentro de un lenguaje de programación o framework, como las APIs de React, Vue.js, jQuery, etc.

Ejemplos

APIs de redes sociales: Permiten a los desarrolladores acceder a los datos de plataformas como Facebook, Twitter, Instagram, etc.
APIs de Google Maps: Permiten la integración de mapas y servicios de localización en aplicaciones web o móviles.
APIs de pago: Como PayPal, Stripe, etc., permiten a los desarrolladores integrar servicios de pago en sus aplicaciones.
